### 2. SQLite原生库修复

**问题**：
```
Failed to load dynamic library 'libsqlite3.so': not found
```

**原因**：`database_manager.dart` 使用 `NativeDatabase.createInBackground()` 但未正确初始化 `sqlite3_flutter_libs` 的原生库。

**修复文件**：`lib/data/database/database_manager.dart`

```dart
// 添加导入
import 'package:sqlite3/sqlite3.dart';
import 'package:sqlite3_flutter_libs/sqlite3_flutter_libs.dart';

// 在创建数据库前初始化原生库
static AppDatabase _createDatabase() {
  final executor = LazyDatabase(() async {
    // 在Android上，确保sqlite3_flutter_libs的原生库被正确加载
    if (Platform.isAndroid) {
      sqlite3.openInMemory().dispose();
    }

    final dbFolder = await getApplicationDocumentsDirectory();
    final file = File(p.join(dbFolder.path, 'body_controller.db'));
    return NativeDatabase.createInBackground(file);
  });
  return AppDatabase(executor);
}
```

### 4. 最终修复方案：drift_sqflite

**问题**：上述SQLite初始化方案仍失败，`sqlite3_flutter_libs 0.6.0+eol` 是废弃包，不包含原生库。

**根本原因**：
- `sqlite3_flutter_libs` 0.6.0+eol 是空包，仅作为兼容性占位
- `sqlite3` 2.x版本不包含Flutter原生库
- APK中缺少 `libsqlite3.so`

**最终解决方案**：使用 `drift_sqflite` + `sqflite`

**修改文件**：

1. `pubspec.yaml`：
```yaml
# 移除
# sqlite3_flutter_libs: ^0.6.0+eol

# 添加
drift_sqflite: ^2.0.0
```

2. `lib/data/database/database_manager.dart`：
```dart
import 'package:drift/drift.dart';
import 'package:drift_sqflite/drift_sqflite.dart';
import 'package:path_provider/path_provider.dart';
import 'package:path/path.dart' as p;
import 'database.dart';

// ...

static AppDatabase _createDatabase() {
  final executor = LazyDatabase(() async {
    final dbFolder = await getApplicationDocumentsDirectory();
    final file = p.join(dbFolder.path, 'body_controller.db');
    return SqfliteQueryExecutor(path: file);
  });
  return AppDatabase(executor);
}
```

**原理**：
- `sqflite` 使用Android系统内置的SQLite（`/system/lib/libsqlite.so`）
- 无需打包额外的原生库，避免兼容性问题

**结果**：训练记录页面正常加载，无错误

## 经验总结

### SQLite方案对比

| 方案 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| `sqlite3_flutter_libs` | 独立SQLite版本 | 0.6.0+eol已废弃 | ❌ |
| `sqlite3` 3.x + Flutter | 新版内置支持 | 依赖冲突多 | ⚠️ |
| `drift_sqflite` + sqflite | 使用系统SQLite，稳定 | 依赖系统版本 | ✅ |

### 教训
1. 遇到原生库问题时，优先考虑使用系统内置库
2. 检查依赖包的维护状态（eol标记）
3. 使用成熟的解决方案（sqflite是Flutter最成熟的SQLite方案）
