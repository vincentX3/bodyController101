 # 个人健身App需求文档（MVP）

**版本**：v1.0  
**目标用户**：算法工程师，同时进行半程马拉松备赛与3分化器械训练  

---

## 1. 产品定位

为个人使用的Android移动端应用，核心解决"双目标训练（跑步+力量）的记录、规划与智能调整"问题。通过AI教练提供数据洞察与可执行计划，支持用户在时间约束下进行动态调整。

---

## 2. 核心功能模块

### 2.1 记录模块（3秒完成记录）

#### 跑步记录
| 字段 | 输入方式 | 说明 |
|-----|---------|------|
| 距离 | 数字键盘 + 快速标签（5/10/15/半马21.1km） | 单位：km，保留1位小数 |
| 时间 | 分、秒两个数字输入框 | 单位：分钟、秒 |
| 配速 | **自动计算展示** | 实时显示，如"5:30/km" |
| 体感 | 1-5星选择 | 可选，默认3星 |
| 日期 | 选择器，默认当天 | 支持补录历史 |

**关键交互**：输入时间后实时计算并显示配速，供用户确认合理性。

#### 力量训练记录
采用"模板+日志"模式，内置3分化模板：

| 分化 | 动作清单 |
|-----|---------|
| 推日 | 杠铃卧推、哑铃卧推、哑铃推举、侧平举、绳索下压、窄距卧推 |
| 拉日 | 引体向上/高位下拉、杠铃划船、杠铃弯举、哑铃弯举、面拉、反向飞鸟 |
| 腿日 | 深蹲、腿举、罗马尼亚硬拉、腿弯举、提踵 |

**记录粒度**：每个动作多组，每组记录重量(kg)、次数、RPE(1-10，可选)。

**交互流程**：选择分化模板 → 勾选今日执行动作 → 逐组录入数据 → 自动计算容量（Σ重量×次数）。

### 2.2 目标模块

| 目标类型 | 设定内容 | 进度追踪 |
|---------|---------|---------|
| 半程马拉松 | 目标日期、目标完赛时间 | 周跑量趋势、最长距离记录、倒计时 |
| 力量目标 | 深蹲/卧推/硬拉目标重量 | 1RM估算曲线、当前最佳记录 |

**目标拆解**：由AI教练在生成计划时自动处理，前端仅展示原始目标与当前进度。

---

### 2.3 AI教练模块（核心差异化）

#### 触发方式
- **主动触发**：任何时刻点击"AI规划"按钮生成分析
- **上下文保留**：支持3轮追问，调整计划

#### 数据洞察（必须包含）
1. **跑量趋势**：周环比%、与目标差距、渐进超负荷判断（增幅>10%标记⚠️）
2. **强度分布**：有氧/乳酸阈值/无氧跑占比
3. **力量进展**：各动作估算1RM变化（Epley公式）、容量周期对比
4. **恢复信号**：连续训练天数、体感评分趋势

#### 输出格式
```
【本周洞察】
• 跑量28km，环比+12% ⚠️ 接近安全上限
• 腿日容量4200kg，较3周前+8% 📈
• 连续5天训练无休息，疲劳累积

【下周计划】
周一：推日力量(50min) + 轻松跑3km@6:00
周三：间歇跑8×800m@4:30/km
周五：拉日力量(55min)
周六：LSD 15km@5:50/km

【调整对话】
用户：下周只有3天能练，周三要加班
AI：理解，压缩方案：周一推日+跑二合一，周六拉腿超级组，跑量减至20km但保强度...
```

#### 技术实现
- 用户自行提供AI API Key（应用内配置）
- 结构化JSON输出 + 自然语言解释

## 3. 技术栈

**仅供参考！根据实际开发确定技术栈**

| 层级 | 选型 | 说明 |
|-----|------|------|
| 跨端框架 | Flutter 3.19 | Vibe Coding友好，热重载快 |
| 状态管理 | Riverpod | 依赖注入清晰，AI生成稳定 |
| 本地存储 | Drift (SQLite) | 类型安全，支持复杂查询 |
| 后端 | Supabase | Auth、云同步、零运维 |
| 图表 | fl_chart | 跑量趋势、力量曲线 |
| AI | OpenAI API | 用户自填API Key，GPT-4o |


## 5. 设计原则

1. **速度优先**：任何记录操作3秒内完成
2. **数据驱动**：所有建议基于实际训练数据，可量化验证
3. **灵活调整**：承认计划赶不上变化，AI支持实时协商
